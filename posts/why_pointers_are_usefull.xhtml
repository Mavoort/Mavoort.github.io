Why Pointers are usefull

Programming 01 January 2021 C++ 

Introduction

For beginners coming from other languages like Python or Octave, pointers in C++ often are reason for confusion. With this post, I want to explain the concept of pointers and why they are usefull.

What is a pointer?

Citing the C++ Primer1:

A pointer is a compound type that points to another type.

What does that mean?

When the program is executed, variables and objects like double, bool or string are stored in the Memory. To each variable belongs a 64-bit address which points to the location within the memory. When the processor has to evaluate or change a variable, it fetches the data at the address of the variable into a register. Then it can do some calculations and move the data back into memory.

This works similar like a library: When you want to read a specific book, you have to get the book from the position of the book number (= the address) and put it back at the same position later.

But this is only theoretical; Let’s look at the Assembly code generated by the compiler. This is a simple program which does nothing more than to increment a variable x.

volatile long x;

int main()
{
x++;
}

And this is the generated Assembly code:

mov rax,QWORD [ptr_x]       ; Move data into register
add rax, 1                  ; Do calculations: Add 1
mov QWORD [ptr_x], rax      ; Move data back to memory

Explanation: The mov instructions does exactly what it sounds like: It moves data, in this case from memory to a register and the other way round. The first operand is the destination, and the second is the source operand. So mov rax, QWORD [ptr_x] moves 8 bytes of data from the memory location with address ptr_x to register rax. Then the processor can do a simple calculation and move the result back into memory, similar to our library example from before.

A Pointer in C++ is nothing more than an object which contains the address of another variable. To assign the address of a specific object to a pointer, we have to use the address-of operator &.

string country;
string* pCountry = &country;

This creates a pointer called pCountry which stores the address of the variable country. If we want to access the data the Pointer points to (not the memory address itself), this can be done with the dereference operator *.

*pCountry = "USA";      // access data it points to

As we see, understanding pointers is important to understand how the concepts of variables and objects work. But are there any real world examples?

Imagine we want to write a function which swaps the content of two strings. The simplest way do achieve this is the following way:

#include <string>
using std::string;

void swap_strings(string&, string&);

int main()
{
string str1 = "Coffee\n", str2 = "Cocoa\n";
swap_strings(str1, str2);
}

void swap_strings(string &str1, string &str2)
{
string tmp = str1;
str1 = str2;
str2 = tmp;
}

Because we want to change the content of the parameters given to the function, we have to declare it call-by-reference. When the function swap_strings is called, it copies the string str1 into the temporary variable tmp. Afterwards, the first string can get the content of str2, and finally tmp is copied back to str2.

Obviously, this has two disadvantages.

  1. We need a temporary variable tmp, which stores a copy of the first string and needs more memory space.
  2. The string objects are copied completely three times in total, which costs a lot of time.

It is much easier to just swap the addresses of both strings by using pointers.

string* pStr1 = &str1;
string* pStr2 = &str2;

void swap_strings(string* &pStr1, string* &pStr2)
{
string* pTmp = pStr1;
pStr1 = pStr2;
pStr2 = pTmp;
}

Let’s compare both implementations with the Google profiler pprof. I created strings of 1kB length each and two functions to swap them, one with and one without using pointers. The functions are called exactly \(10^8\) times.

pprof swap_strings /tmp/prof.out --ignore="std|__gnu|_init"

As expected, the function which uses pointers is significantly faster.

Conclusion: We use pointers because it’s easier to give someone an address to your home than to give a copy of your home to everyone.2
